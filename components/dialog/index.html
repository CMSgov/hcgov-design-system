    Modal dialog

    Jump to guidance View related guidance in the CMS Design System

    The dialog component can be used to focus a user's attention on a single piece of content, without taking them to a new screen. Please use with caution; view our guidance for more details.

    class Example extends React.PureComponent {
  constructor(props) {
    super(props);

    this.state = {
      showModal: false,
    };
  }

  hideModal() {
    this.setState({ showModal: false });
  }

  showModal() {
    this.setState({ showModal: true });
  }

  render() {
    return (
      <div id="App" style={{ minHeight: 300 }}>
        <Button onClick={() => this.showModal()} size="big" variation="success">
          Click to show modal
        </Button>

        {this.state.showModal && (
          <Dialog
            onExit={() => this.hideModal()}
            getApplicationNode={() => document.getElementById('App')}
            heading="Dialog heading"
            actions={[
              <button className="ds-c-button ds-c-button--primary" key="primary">
                Dialog action
              </button>,
              <button
                className="ds-c-button ds-c-button--transparent"
                key="cancel"
                onClick={() => this.hideModal()}
              >
                Cancel
              </button>,
            ]}
          >
            Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed accumsan diam vitae metus
            lacinia, eget tempor purus placerat.
          </Dialog>
        )}
      </div>
    );
  }
}

ReactDOM.render(<Example />, document.getElementById('js-example'));

    Props

    React Properties Documentation
    NameTypeDefaultDescription
    alertbool

    If true, the modal will receive a role of alertdialog, instead of its default dialog. The alertdialog role should only be used when an alert, error, or warning occurs.

    applicationNodecustom

    Provide a DOM node which contains your page's content (which the modal should render outside of). When the modal is open this node will receive aria-hidden="true". This can help screen readers understand what's going on. Also see getApplicationNode.

    actionsstring, number, element, or array

    Buttons or other HTML to be rendered in the "actions" bar at the bottom of the dialog.

    actionsClassNamestring

    Additional classes to be added to the actions container.

    ariaCloseLabelstring'Close modal dialog'

    Aria label for the close button

    children

    Required

    		string, number, element, or array

    The modal's body content

    classNamestring

    Additional classes to be added to the root dialog element.

    closeButtonSize'small', 'big'

    Size of the close button. See Button component

    closeButtonVariationstring'transparent'

    Variation string to be applied to close button component. See Button component

    closeTextstring, number, element, or array'Close'

    For internationalization purposes, the text for the "Close" button must be passed in as a prop.

    escapeExitDisabledboolfalse

    Disable exiting the dialog when a user presses the Escape key.

    getApplicationNodefunc

    Same as applicationNode, but a function that returns the node instead of the node itself. The function will not be called until after the component mounts, so it's safe to use browser globals and refer to DOM nodes within it (e.g. document.getElementById(..))

    headerClassNamestring

    Additional classes to be added to the header, which wraps the heading and close button.

    headingstring, node

    The Dialog's heading, to be rendered in the header alongside the close button.

    onExitfunc

    A method to handle the state change of exiting (or deactivating) the modal. It will be invoked when the user presses Escape, or clicks outside the dialog (if underlayClickExits=true).

    size'narrow', 'wide', 'full'
    underlayClickExitsboolfalse

    Enable exiting the dialog when a user clicks the underlay.
